## 一、核心架构

### 1.1 四个报告的触发条件

| 时段 | 触发词 | 报告类型 |
|------|--------|---------|
| 盘前 9:00 | 盘前分析, 盘前, 隔夜, pre-market | 市场全景报告 |
| 集合竞价 9:25 | 竞价, 集合竞价, 竞价分析, 开盘前 | 竞价解读报告 |
| 早盘 10:00 | 早盘, 盘中, 盘中分析, 上午 | 盘中策略报告 |
| 复盘 15:00 | 复盘, 收盘, 收盘分析, 复盘报告, 盘后 | 收盘复盘报告 |
| 个股 (任意时间) | 分析股票, 个股分析, 研报, 股票报告, [代码] | 个股深度研报 |

当用户说"看看今天市场""今天怎么样"自动判断当前时间，选择最接近的报告时段。

### 1.2 时间判断逻辑

```
当前时间 < 9:15   → 盘前分析（含隔夜美股+昨日市场回顾）
9:15 ~ 9:40       → 集合竞价解读
9:40 ~ 11:35      → 早盘分析
11:35 ~ 14:00     → 午间休市提示（可简单概述上午情况）
14:00 ~ 16:00     → 收盘复盘
16:00 ~ 次日 9:00 → 盘前分析（含今日复盘+明日展望）
周末/节假日       → 周度回顾+下周前瞻
```

### 4.2 分析框架

```
## 个股深度分析框架

### 一、公司基本面
- 所属行业、主营业务（各业务占比）
- 行业地位（龙头/跟随/边缘）
- 实控人/管理层背景
```

## 四-A、防幻觉与数据质量铁律（必读）

> ⚠️ **本技能所有数据输出必须经过浏览器核验。** 以下是不可违反的铁律。

### 铁律1：看见标签≠看见数据

浏览器页面上出现"韩国KOSPI""日经225"等标签，**不等于**这些市场今天有交易数据。
- 如果页面没有显示实际价格数字 → 说明市场休市 → **绝对不能自己编一个数字**
- 必须确认数据确实加载出来后，才能引用

### 铁律2：每个数字必须有来源

报告中的每个指数涨跌幅、涨停数、资金金额、公司估值，必须能追溯到：
- 浏览器截图上的可见数字，或
- web_search 返回的明确数据

**不允许：** 模糊估计、"大概""预估"、看到标签就脑补数字

### 铁律3：市场日历先行

引用任何境外市场（美股、日股、韩股、港股等）前，先确认今天是否开市：
```
web_search搜索："韩国 股市 今天 休市 2026年5月XX日"
```
只有确认开市后才能引用该市场数据。

### 铁律4：数据一致性检查

报告写完后，通读一遍检查：
- 数字合理吗？（休市日不可能有波动）
- 数量级对吗？（15万亿 vs 1.75万亿）
- 前后矛盾吗？（涨停数写6家但下文提到8只个股）

### 铁律5：宁缺毋滥

拿不准的数据，**宁可不写也不要编**。
- 可以写"数据待核实"
- 可以省略不确定的数据点
- 编造的数据会毁了整份报告的可信度

## 五、交叉验证机制

### 5.1 数据验证等级

```
等级A（高可信）：以下数据来源互相一致
- stock_data.py AKShare 输出 vs 浏览器东方财富页面
- 多个搜索结果的相同数据

等级B（中等）：2个独立来源一致
- 搜索结果数量≥2且数据一致
- 或 单个可靠来源（东方财富/同花顺）获取

等级C（低可信/需标注）：仅1个来源数据
- 必须标注"来源：{来源}，可能不完整"
- 带有"据传""疑似"等不确定性标签
```

### 5.2 数据核验流程

```
1. 行情数据核验
   stock_data.py indexes/updown/limits → 浏览器东方财富开盘啦验证

2. 板块数据核验
   collector.py sector → 浏览器东方财富板块资金页验证
   不一致时：以浏览器页面为准，标注stock_data偏差

3. 个股价格核验
   至少2个来源：东方财富 + 同花顺/腾讯
   不一致超过1%时：search/refresh，标注来源

4. 财务数据核验
   搜索得到财报摘要 → 如有官网财报链接则核实
   注意报告期（季度/半年/全年）

5. 新闻/消息核验
   至少2个独立来源一致才确认
   单一来源消息标注"引用自{来源}"
```

### 5.3 自动验证脚本

```bash
# 数据自洽性检查
python scripts/verifier.py --check self-consistent

# 交叉验证（Browser + API）
python scripts/verifier.py --cross-validate

# 生成验证报告
python scripts/verifier.py --report
```

### 七、跨平台注意事项

| 项目 | Linux | macOS | Windows |
|------|-------|-------|---------|
| Python命令 | `python3` 或 `python` | `python3` 或 `python` | 只能用 `python` |
| 路径分隔符 | `/` | `/` | `\\`（脚本自动适配）|
| 用户目录 `~` | `/home/用户名` | `/Users/用户名` | `C:\\Users\\用户名` |
| OpenClaw安装 | `curl ... install.sh` | `curl ... install.sh` | `iwr ... install.ps1` |

> 所有脚本内部使用 `sys.executable` 替代硬编码的 `python3`，
> 因此无论在哪个平台，脚本调用都会自动使用当前运行的Python解释器，无需手动配置。
